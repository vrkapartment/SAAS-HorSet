# 🏢 App Corporate Identity (CI) & Adaptive Brand Guidelines

เอกสารฉบับนี้คือคู่มืออัตลักษณ์สากลของแบรนด์ (Corporate Identity) และระบบ Adaptive Design โปรดใช้กฎเกณฑ์เหล่านี้เป็นแนวทางหลักในการควบคุมสี (Colors), ฟอนต์ (Typography), และการเลือกสลับชุดโครงสร้างเลย์เอาต์ (Adaptive Layout Blocks) ให้เหมาะสมกับขนาดหน้าจอของผู้ใช้งานอย่างเคร่งครัด เพื่อให้แอปพลิเคชันมีความน่าเชื่อถือ เป็นมืออาชีพ และทำงานได้อย่างมีประสิทธิภาพสูงสุด

---

## 👁️ 1. Brand Identity & Personality (ตัวตนและอารมณ์ของแบรนด์)

แอปพลิเคชันนี้ถูกวางตัวตนให้เป็นระบบบริหารจัดการ (SaaS) ที่มีความเป็นมืออาชีพสูง เข้าถึงง่าย และชาญฉลาด ดังนั้น อารมณ์ในการออกแบบ (Mood & Tone) จะต้องยึดหลัก 3 ข้อดังนี้:
1. **Trustworthy (น่าเชื่อถือ):** ใช้โครงสร้างที่มั่นคง สะอาดตา และระบบสีที่ให้ความรู้สึกปลอดภัย
2. **Efficient (ทรงประสิทธิภาพ):** ทุกองค์ประกอบต้องออกแบบมาเพื่อความรวดเร็วในการใช้งาน ไม่รกรุงรัง
3. **Modern & Premium (ทันสมัยและดูแพง):** ใช้พื้นที่ว่าง (Whitespace) และเส้นสายที่บางเบา ทันสมัย

---

## 🖥️ 2. Adaptive Design System (ระบบการออกแบบสลับเลย์เอาต์ตามอุปกรณ์)

ระบบนี้ไม่ได้ใช้เพียงแค่การยืดหดธรรมดา (Responsive) แต่ให้ใช้หลักการ **Adaptive Design** คือการเตรียม "บล็อกส่วนประกอบ (Component Blocks) ที่คิดมาเพื่ออุปกรณ์นั้น ๆ โดยเฉพาะ" แล้วสั่งสลับชุดการแสดงผลตามขนาดหน้าจอดังนี้:

### 2.1 Viewport Breakpoints (จุดตัดขนาดหน้าจอ)
*   **Mobile View (Mobile/Tablet Compact):** หน้าจอขนาดต่ำกว่า `768px` (ใช้สไตล์อินพุตและปุ่มขนาดใหญ่สำหรับนิ้วสัมผัส)
*   **Desktop View (Laptop/Desktop Wide):** หน้าจอขนาดตั้งแต่ `768px` ขึ้นไป (ใช้สไตล์เน้นข้อมูลหนาแน่นและแสดงผลแบบตาราง/คอลัมน์คู่)

### 2.2 Adaptive Structural Components (การสลับบล็อกโครงสร้างหลัก)

| ส่วนประกอบหลัก | 📱 บล็อกสำหรับ Mobile View (< 768px) | 💻 บล็อกสำหรับ Desktop View (≥ 768px) |
| :--- | :--- | :--- |
| **Navigation** | **Bottom Navigation Bar:** แถบเมนูด้านล่างสุดยึดติดหน้าจอ 4-5 เมนูหลัก เพื่อให้นิ้วโป้งกดง่าย | **Fixed Left Sidebar:** แถบเมนูด้านข้างซ้ายแบบเต็มความสูง แสดงรายการลิงก์และไอคอนครบถ้วน |
| **Data Display** | **Card-based List View:** แตกแถวข้อมูลออกมาเป็นกล่องการ์ดแนวตั้ง 1 ใบต่อ 1 รายการข้อมูล | **Data Table View:** แสดงผลเป็นตารางสากล `<table>` มีคอลัมน์ครบถ้วน พร้อมระบบแบ่งหน้า (Pagination) |
| **Form Layout** | **Single Column Scroll:** ฟอร์มกรอกข้อมูลแถวเดี่ยวเรียงจากบนลงล่าง ขนาด Input สูงเด่น | **Multi-column Grid:** จัดวางอินพุตเป็น 2-4 คอลัมน์ต่อแถว เพื่อใช้พื้นที่หน้าจอกว้างให้คุ้มค่า |
| **Action Items** | **Sticky Bottom Action:** ปุ่มยืนยัน/เซฟข้อมูลหลัก จะลอยติดอยู่ด้านล่างสุดของจอเสมอ | **Top-Right Inline Action:** ปุ่มจัดการข้อมูลจะอยู่มุมบนขวาของกล่องคอนเทนต์นั้น ๆ |

## 🎨 3. Color System & Token (ระบบสีหลักและสีรอง)

โปรดใช้โค้ดสีเหล่านี้ (ผ่านมาตรฐาน Web Accessibility และรองรับทั้งสองโหมด) ในการจัดแต่งสไตล์ผ่าน Tailwind CSS อย่างเคร่งครัด:

### 3.1 Brand Primary & Secondary Colors
*   **Primary Blue (สีน้ำเงินหลัก - สื่อถึงความน่าเชื่อถือและความเป็นมืออาชีพ):**
    *   Light Mode: `text-blue-600` / `bg-blue-600` (HEX: `#2563eb`)
    *   Dark Mode: `dark:text-blue-500` / `dark:bg-blue-500` (HEX: `#3b82f6`)
*   **Secondary Indigo (สีม่วงน้ำเงิน - ใช้สำหรับเน้นจุดเด่น หรือปุ่มสำคัญรอง):**
    *   Light Mode: `text-indigo-600` / `bg-indigo-600` (HEX: `#4f46e5`)
    *   Dark Mode: `dark:text-indigo-400` / `dark:bg-indigo-400` (HEX: `#818cf8`)

### 3.2 Functional Colors (สีสถานะ)
*   **Success (สีเขียว - รายการสำเร็จ/จ่ายเงินแล้ว/ห้องว่าง):** `emerald-600` | พื้นหลังอ่อน `bg-emerald-50`
*   **Warning (สีเหลือง/ส้ม - ค้างจ่าย/รอดำเนินการ):** `amber-500` | พื้นหลังอ่อน `bg-amber-50`
*   **Danger/Alert (สีแดง - ลบข้อมูล/ยกเลิกสัญญา/อันตราย):** `red-600` | พื้นหลังอ่อน `bg-red-50`

### 3.3 System Neutral Colors (สีพื้นหลังและตัวอักษร)
*   **Background:** ด้านนอกใช้ `bg-slate-50` / `dark:bg-slate-900` | บนการ์ดข้อมูลใช้ `bg-white` / `dark:bg-slate-800`
*   **Text Colors:** ตัวอักษรหลักใช้ `text-slate-900` / `dark:text-slate-50` | ตัวอักษรรองใช้ `text-slate-500` / `dark:text-slate-400`

## ✍️ 4. Typography & Hierarchy (ระบบฟอนต์และสัดส่วน)

### 4.1 Font Family (ตระกูลฟอนต์)
*   ให้ใช้ฟอนต์สไตล์ไร้หัวที่ดูโมเดิร์นและสะอาดตา (Sans-serif) เช่น `Inter` สำหรับภาษาอังกฤษ ควบคู่กับ `Noto Sans Thai` หรือ `Sarabun` (เวอร์ชันไม่มีหัวสำหรับหน้าจอแอปพลิเคชัน)
*   Tailwind Class: `font-sans`

### 4.2 Font Scaling ตามอุปกรณ์ (Adaptive Typography)
*   **Main Title:** บน Mobile ใช้ `text-2xl` | บน Desktop ใช้ `text-3xl font-bold tracking-tight`
*   **Section Header:** บน Mobile ใช้ `text-lg` | บน Desktop ใช้ `text-xl font-semibold`
*   **Body Text:** ใช้ขนาดเท่ากันทุกอุปกรณ์เพื่อความอ่านง่าย `text-sm font-normal`

## 📐 5. UI Elements Standard (มาตรฐานองค์ประกอบภาพ)

*   **Border Radius (ความโค้งมน):** ปุ่มและช่องอินพุตใช้ `rounded-xl` | การ์ดข้อมูลและ Modal ใช้ `rounded-2xl`
*   **Icons (ระบบไอคอน):** ใช้ไอคอนสไตล์เส้นบาง (Outline) จากชุดเดียวกันทั้งแอปพลิเคชัน (แนะนำ `Lucide React` หรือ `Heroicons`) ขนาดมาตรฐานคือ `w-5 h-5`

---

## 💬 6. Brand Voice & Copywriting (ภาษาและคำพูดในแอป)

*   **สุภาพและเป็นมืออาชีพ:** ไม่ใช้ภาษาแสลง หรือคำที่เป็นทางการเกินไปจนเข้าใจยาก
*   **กระชับและชัดเจน (Action-Oriented):** บอกให้ผู้ใช้รู้ทันทีว่าเกิดอะไรขึ้น และต้องทำอะไรต่อ
*   **คำปุ่มกด (Button Labels):** ใช้คำกริยาที่สั้นและตรงตัว เช่น "บันทึกข้อมูล", "ยกเลิก", "ส่งบิลค่าเช่า"

## 🛠️ 7. AI Agent Implementation Instructions (ข้อบังคับสำหรับ AI)

เมื่อทำการสร้างหรือแก้ไข Component ใด ๆ ในโปรเจกต์นี้:
1.  **Strict Adaptive Check:** ตรวจสอบโครงสร้างเลย์เอาต์เสมอ โดยใช้กลไกการซ่อน/แสดงผล หรือแยกชิ้น Component ระหว่าง Mobile และ Desktop ตามกฎใน **PART 2** (เช่น ใช้คลาส `block md:hidden` สำหรับบล็อกโมบายล์ และ `hidden md:block` สำหรับบล็อกเดสก์ท็อป)
2.  **Color & Token Discipline:** ต้องใช้ Token สีจาก **PART 3** ควบคู่กับการรองรับ `dark:` โหมดในทุกไฟล์
3.  **No Dynamic Resizing Overlap:** ห้ามใช้วิธีการยืดขนาดตารางเดสก์ท็อปให้หดเหลือตัวกระจิ๋วบนโมบายล์เด็ดขาด หากขนาดจอเข้าสู่สเกลโมบายล์ ต้องสลับไปใช้โครงสร้าง Card-based ทันที

# คู่มือการจัดการและอัปเดตแบบฟอร์มภาษี ภ.ง.ด. 90 / 94 (PDF Form Filling Guide)

> เอกสารนี้จัดทำขึ้นเพื่อบันทึกวิธีการทำงานของระบบกรอกแบบฟอร์มภาษี และเป็นคู่มือสำหรับนักพัฒนาในการอัปเดตเทมเพลตไฟล์ PDF เมื่อกรมสรรพากรประกาศปรับปรุงแบบฟอร์มในรอบปีภาษีใหม่

## 🏗️ สถาปัตยกรรมการทำงานปัจจุบัน (Current Architecture)

ระบบกรอกฟอร์ม PDF ของ HorSet พัฒนาด้วย Next.js และทำงานบนฝั่ง Client-Side (Browser) ทั้งหมด โดยใช้ไลบรารี `pdf-lib` ร่วมกับ `@pdf-lib/fontkit` ในการกรอกข้อมูลลงในฟิลด์ PDF ต้นฉบับแบบอัตโนมัติ

### ส่วนประกอบที่สำคัญในระบบ:
1. **ไฟล์ PDF ต้นแบบ (PDF Templates)**: 
   * จัดเก็บที่: `public/templates/`
   * ไฟล์ปัจจุบัน: `201267PIT90.pdf` (ภ.ง.ด. 90) และ `250668PIT94.pdf` (ภ.ง.ด. 94)
   * *คุณสมบัติสำคัญ*: ต้องเป็นเอกสาร PDF แบบกรอกได้ (Interactive PDF Form/AcroForm)
2. **ฟอนต์ภาษาไทย (Thai Font)**:
   * ดึงมาจาก: CDN (Google Fonts - Sarabun Regular)
   * หน้าที่: ใช้ฝังลงในเอกสาร PDF เพื่อให้ช่องฟอร์มแสดงภาษาไทยและตัวเลขไทย/อารบิกได้อย่างถูกต้อง (สระไม่จมหรือลอย)
3. **ตัวช่วยกรอกข้อมูล (PDF Helper)**:
   * ไฟล์หลัก: [src/lib/pdfHelper.ts](file:///G:/SAAS%20HorSet/src/lib/pdfHelper.ts)
   * ทำหน้าที่โหลดเทมเพลต, โหลดฟอนต์, กรอกข้อมูลลงฟิลด์ตามรหัสฟิลด์ (Form Field Names) และคำนวณสัดส่วนรายได้ครึ่งปี/เต็มปี

## 🛠️ ขั้นตอนการอัปเดตเทมเพลต PDF ในปีถัดไป (Step-by-Step Guide for Next Year)

เมื่อกรมสรรพากรประกาศแบบฟอร์ม ภ.ง.ด. ใหม่ ให้ดำเนินงานตามขั้นตอนดังนี้:

### ขั้นตอนที่ 1: ดาวน์โหลดไฟล์ต้นแบบจากกรมสรรพากร
1. ไปที่เว็บไซต์สรรพากร [RD E-Filing](https://www.rd.go.th/) หรือเมนูแบบแสดงรายการภาษี
2. ดาวน์โหลดเอกสารแบบแสดงรายการภาษี ภ.ง.ด. 90 หรือ 94 ชนิดที่เป็น **PDF แบบกรอกข้อมูลได้ (Interactive Form)**
3. นำมาบันทึกไว้ในโปรเจคที่โฟลเดอร์ `public/templates/` (เช่น `public/templates/2027PIT90.pdf`)

### ขั้นตอนที่ 2: รันสคริปต์สแกนหาชื่อฟิลด์ใน PDF (Field Name Extraction)
แบบฟอร์ม PDF ของสรรพากรแต่ละปีจะมีชื่อฟิลด์ (เช่น `Text7.0`, `Text80.0`) เปลี่ยนแปลงไป นักพัฒนาจำเป็นต้องนำไฟล์ PDF ใหม่มาสแกนเพื่อหาว่าฟิลด์ของแต่ละข้อมูลมีชื่อรหัสว่าอะไร

รันคำสั่ง Node.js ด้านล่างนี้เพื่อสกัดรายชื่อฟิลด์ทั้งหมดและข้อความคำอธิบาย (Tooltip) ออกมาเป็นไฟล์ข้อความ:

```javascript
// บันทึกสคริปต์นี้เป็น extract-fields.js แล้วรันด้วยคำสั่ง: node extract-fields.js
const { PDFDocument } = require('pdf-lib');
const fs = require('fs');

async function extractFields(pdfPath, outputPath) {
  try {
    const pdfBytes = fs.readFileSync(pdfPath);
    const pdfDoc = await PDFDocument.load(pdfBytes);
    const form = pdfDoc.getForm();
    const pages = pdfDoc.getPages();
    
    const lines = form.getFields().map(f => {
      const name = f.getName();
      const type = f.constructor.name;
      
      // ดึง Tooltip/Description ของฟิลด์ (ถ้ามี) เพื่อช่วยให้รู้ว่ามันคือฟิลด์อะไร
      const dict = f.acroField.dict;
      const tu = dict ? dict.get(require('pdf-lib').PDFName.of('TU')) : null;
      const label = tu ? tu.decodeText() : '(ไม่มีคำอธิบาย)';
      
      // ค้นหาว่าฟิลด์นี้อยู่หน้าไหนของเอกสาร
      let pageNum = 'Unknown';
      if (f.acroField.getWidgets().length > 0) {
        const ref = f.acroField.getWidgets()[0].dict.get(require('pdf-lib').PDFName.of('P'));
        if (ref) {
          pageNum = pages.findIndex(p => p.ref === ref) + 1;
        }
      }
      
      return `Page ${pageNum} | Field: ${name} (${type}) -> คำอธิบาย: ${label}`;
    });
    
    fs.writeFileSync(outputPath, lines.join('\n'), 'utf8');
    console.log(`สแกนฟิลด์สำเร็จ! บันทึกข้อมูลไปที่: ${outputPath}`);
  } catch (error) {
    console.error('เกิดข้อผิดพลาด:', error);
  }
}

// ตัวอย่างการเรียกใช้งาน (ปรับเปลี่ยนชื่อไฟล์ PDF ตามจริง)
extractFields('public/templates/2027PIT90.pdf', 'fields_pit90_new.txt');
```

> **คำแนะนำเพิ่มเติม**: เปิดดูไฟล์ผลลัพธ์ `fields_pit90_new.txt` แล้วค้นหาคีย์เวิร์ดภาษาไทย เช่น "เลขประจำตัว", "ชื่อ", "นามสกุล", "เช่า", "หักค่าใช้จ่าย" เพื่อระบุรหัสของฟิลด์ที่ต้องการนำมาใช้

### ขั้นตอนที่ 3: จับคู่ฟิลด์และแก้ไขโค้ด (Update Field Mappings)
แก้ไขไฟล์ [src/lib/pdfHelper.ts](file:///G:/SAAS%20HorSet/src/lib/pdfHelper.ts) โดยเปลี่ยนพาธเทมเพลตและจับคู่ฟิลด์ให้ตรงกับเอกสารใหม่:

```typescript
// 1. เปลี่ยนชื่อไฟล์เทมเพลตใหม่ในส่วนแรก
const templateUrl = type === "90"
  ? "/templates/2027PIT90.pdf" // ปรับเป็นไฟล์ของปีใหม่
  : "/templates/2507PIT94.pdf"
```

ปรับชื่อฟิลด์ตามที่สแกนได้ในบล็อก `type === "90"` และบล็อก `else` (ภ.ง.ด. 94):

```diff
// ตัวอย่าง: หากสรรพากรเปลี่ยนรหัสฟิลด์ชื่อผู้เสียภาษี
- setField("Text7.0", data.firstName)
+ setField("Text9.0", data.firstName) // แก้ไขรหัสให้ตรงกับเวอร์ชันใหม่
```

## 💡 ข้อแนะนำและแนวทางการตรวจสอบปัญหา (Troubleshooting & Best Practices)

1. **ปัญหาฟอนต์ภาษาไทยไม่ขึ้น**:
   * หากทดสอบในโหมด Local แล้วกรอกภาษาไทยแต่ PDF ไม่แสดงผล (เป็นช่องสี่เหลี่ยมหรือว่างเปล่า) ให้ตรวจสอบว่าได้เรียกใช้งาน `form.updateFieldAppearances(customFont)` ก่อนสั่งเซฟไฟล์ PDF แล้วหรือยัง (ไลบรารี `pdf-lib` ต้องการฟอนต์เพื่อสร้าง Object Appearance ของฟิลด์ใหม่)
2. **วิธีการเช็คพิกัด XY สำหรับข้อความพิเศษ**:
   * หากต้องวาดรายละเอียดพิเศษนอกเหนือจากฟิลด์ฟอร์ม (เช่น ลายน้ำ หรือ เบอร์โทรใน ภ.ง.ด. 90) สามารถรันโค้ด `drawText(text, x, y, size)` ได้
   * พิกัดมุมซ้ายล่างของหน้ากระดาษคือ `(x: 0, y: 0)` และมุมขวาบนมักจะมีค่าประมาณ `(x: 595, y: 842)` (ขนาดมาตรฐาน A4)
3. **การตรวจสอบผลลัพธ์ผ่าน Console**:
   * ฟังก์ชัน `setField` ปัจจุบันมี Try-Catch Block ครอบไว้ หากกรอกชื่อฟิลด์ผิดหรือสรรพากรลบฟิลด์นั้นทิ้ง ระบบจะไม่แครช แต่จะแจ้งเตือนใน Console Log ของหน้าเว็บบราวเซอร์ว่า `ไม่สามารถกรอกฟิลด์ [ชื่อฟิลด์]: Error...` ช่วยให้นักพัฒนาทราบจุดบกพร่องได้อย่างรวดเร็ว
